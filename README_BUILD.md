# Сборка и распространение

Руководство для разработчика по сборке дистрибутивов Вехи.

---

## Инструменты

- **Node.js** v18 или выше — https://nodejs.org (LTS)
- **electron-builder** — устанавливается автоматически через `npm install`
- Для сборки `.exe` под Windows рекомендуется собирать на Windows-хосте
- Для `.AppImage` под Linux — на Linux или через CI

---

## Подготовка

```bash
npm install
```

Первая установка загружает Electron (~120 МБ) и electron-builder. Последующие — используют кэш npm.

Перед сборкой убедитесь, что приложение запускается корректно:

```bash
npm start
```

---

## Сборка

### Windows — NSIS-установщик

```bash
npm run build
```

Результат в `dist/`:
```
dist/
  Веха Setup 1.0.0.exe   ← установщик для пользователей
  win-unpacked/          ← портативная версия без установки
```

### Linux — AppImage

```bash
npm run build:linux
```

Результат в `dist/`:
```
dist/
  Веха-1.0.0.AppImage
```

### Оба таргета за один запуск

```bash
npm run build:all
```

## Установка у пользователя

**Windows.** Запустить `Веха Setup 1.0.0.exe`. Установщик предложит выбрать папку, создаст ярлыки в меню Пуск и на рабочем столе. Данные хранятся в `%APPDATA%\vekha` и не удаляются при переустановке.

**Linux.** Сделать AppImage исполняемым и запустить:

```bash
chmod +x Веха-1.0.0.AppImage
./Веха-1.0.0.AppImage
```

На Astra Linux и ряде других дистрибутивов требуется пакет `libfuse2`:

```bash
sudo apt install libfuse2
```

Если среда не поддерживает FUSE (например, контейнеры), добавить флаг `--no-sandbox`:

```bash
./Веха-1.0.0.AppImage --no-sandbox
```

## Иконка приложения

В `package.json` прописана иконка:

```json
"win":   { "icon": "src/assets/logo.jpg" },
"linux": { "icon": "src/assets/logo.png" }
```

Для замены: положить новый файл в `src/assets/` и обновить пути в `package.json`. Для Windows рекомендуется формат `.ico` (256×256 px).

---

## Выпуск новой версии

1. Обновить поле `"version"` в `package.json` (например, `"1.1.0"`)
2. Собрать: `npm run build` / `npm run build:linux`
3. Передать новый дистрибутив пользователям

При установке поверх предыдущей версии база данных сохраняется. Миграции схемы выполняются автоматически при первом запуске новой версии.

## Возможные проблемы

**`npm install` не завершается или ошибка сети**

Попробуйте явно указать реестр:
```bash
npm install --registry https://registry.npmjs.org
```

**Ошибка NSIS при сборке под Windows**

- Убедитесь, что антивирус не блокирует `electron-builder`
- Запустите терминал с правами администратора

**AppImage не запускается на Astra Linux**

Проверьте наличие `libfuse2` (см. выше). Если проблема сохраняется — попробуйте флаг `--no-sandbox`.

**Иконка в собранном .exe остаётся стандартной Electron**

Убедитесь, что файл иконки находится по пути, указанному в `package.json`, и пересоберите. Кэш electron-builder иногда нужно сбросить вручную: удалите папку `dist/` перед повторным запуском сборки.
